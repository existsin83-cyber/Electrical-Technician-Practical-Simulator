## 🚨 3. 현재 식별된 심각한 결함 및 분석 (Critical Bugs)

### 결함 A: 도면 대비 네온 오버레이(전선)의 좌표 어긋남 현상
* **증상**: 
  * 렌더링된 회로도 도면과 오렌지색/적색 네온 전류 가이드라인의 위치가 일치하지 않고 크게 어긋남.
  * 하단의 코일/램프 핫스팟(원형 노드)과 전선들이 전반적으로 Y축 방향으로 아래로 쏠려 있으며, 도면 경계를 벗어나기도 함. X축 간격 역시 도면 우측으로 갈수록 오차가 벌어짐.
* **원인 분석**:
  * PDF.js가 렌더링하는 Canvas의 실제 픽셀 스케일링 팩터(`viewport.scale`)와 SVG의 `viewBox` 배율 간에 불일치가 존재하거나, `task01.js`에 작성된 실측 좌표 `[X, Y]`의 기준 해상도 포인트가 PDF 원본의 Viewport 사이즈(842x595 포인트)와 완벽히 정렬되지 않았습니다.
  * 컨테이너 크기 변경에 따른 스케일 보정 수식의 보완이 필요합니다.

### 결함 B: 자동 모드 시퀀스 및 기구 동작 논리 모순
* **증상**:
  * 셀렉터 스위치를 **자동(A)**으로 두고 FLS 수위 감지를 **수위 가능(쇼트)**으로 설정했을 때, 우측 모니터 상에서 `Relay X`와 `Timer T`는 비활성화(회색) 상태인데, `MC1`과 `RL`만 동작(붉은색)하는 비논리적 현상이 발생함.
* **원인 분석**:
  * `engine.js` 내의 BFS/DFS 그래프 탐색 시 `SS`의 모드가 `'A'`일 때 탐색 큐에서 자동 모드 분선(`W_SS_A`, `W_FLS_A_IN`, `W_FLS_A_OUT`)을 타고 `X_POWER` 코일에 전위가 전달되는 흐름에 결함이 있거나,
  * 반대로 릴레이 X가 소자되었음에도 `X_A2` 접점이나 `MC1_POWER` 코일이 통전된 것으로 판단되는 그래프 노드 연결성(`task01.js`의 `wires` 매핑) 설계 오류가 존재합니다.
  * 회로도 도면의 순시/한시 접점 및 인터록(Interlock) 논리를 그래프 연결성 관점에서 전면 재검토해야 합니다.

## Claude (Opus 4.8)의 분석 및 개선 의견 (2026-06-10)

코드를 직접 읽고(App.jsx / PdfCanvas.jsx / engine.js / task01.js) 두 결함의 근본 원인을 특정했습니다. 아직 코드는 수정하지 않았고, 먼저 분석/계획을 보고합니다.

### 결함 A: 좌표 어긋남 — 원인 확정 (구조적 박스 불일치)

증상(전선/노드가 우측·하단으로 갈수록 더 벌어지며 쏠림)은 `viewport.scale` 수식 문제가 아니라 **Canvas 표시 박스와 SVG 오버레이 박스가 서로 다른 크기**라서 발생합니다. `PdfCanvas.jsx`에서:

1. 컨테이너에 `className="glass-panel p-2"`로 **내부 패딩(약 8px)** 이 있습니다.
2. 스케일 계산에 `containerRef.current.clientWidth`를 씁니다. `clientWidth`는 **패딩을 포함**하므로, `canvas.width`(내부 픽셀)는 패딩 포함 폭으로 잡힙니다 (`PdfCanvas.jsx:565-567`).
3. 그런데 canvas는 CSS `width:100%`로 **콘텐츠 박스(=clientWidth − 좌우 16px)** 에 맞춰 축소 표시됩니다 (`PdfCanvas.jsx:654`).
4. 반면 SVG는 `width:${dimensions.width}px` / `height:${dimensions.height}px` 즉 **축소 전 canvas 픽셀 크기 그대로**, 그리고 `top:8px left:8px`로 하드코딩되어 얹힙니다 (`PdfCanvas.jsx:661-668`).

결과적으로 SVG가 실제로 보이는 canvas보다 가로로 ~16px, 세로로 그 비율만큼 더 크게 그려집니다. viewBox는 842×595 고정이므로 동일 포인트가 더 넓은 박스에 매핑 → **우측으로 갈수록 X 오차 증가, 아래로 갈수록 Y 오차 증가**라는 보고된 증상과 정확히 일치합니다.

**해결 방향 (권장)**: 스케일 수식을 만지는 대신 박스를 일치시킵니다.
- canvas와 SVG를 동일한 relative wrapper에 넣고, SVG를 `position:absolute; inset:0; width:100%; height:100%`로 두고 `viewBox="0 0 842 595"`만 유지 → 좌표 매핑이 해상도/배율과 무관하게 자동 정렬됩니다. `top:8px/left:8px`, 픽셀 `dimensions` 하드코딩 제거.
- 스케일 기준은 패딩이 포함된 `clientWidth`가 아니라 콘텐츠 폭을 쓰거나, 패딩(`p-2`)을 안쪽이 아닌 바깥 래퍼로 옮깁니다.
- (선택) 선명도를 위해 `devicePixelRatio` 반영.

### 결함 B: 자동 모드 논리 모순 — 원인은 "엔진 모델 자체"

현재 `engine.js`는 **L1(FUSE_L1)에서 BFS로 도달 가능하기만 하면 코일을 여자**시킵니다(`engine.js:801-867`). 두 가지 구조적 문제가 있습니다.

1. **귀로(L2/중성선) 검증이 없음**: 코일의 반대 단자가 L2로 닫혀 폐회로를 이루는지 확인하지 않고, "L1에서 닿으면 켜짐"으로 판정합니다. 실제 시퀀스는 양 단자 폐회로가 성립해야 동작합니다.
2. **코일이 도체처럼 통과됨**: BFS가 코일 노드를 만나도 막지 않고 그 너머 노드로 계속 전위를 전파합니다. 코일은 부하(종단)인데 통과 경로로 취급되어 의도치 않은 노드가 살아납니다.

여기에 더해, 자기유지 피드백(`prevActiveCoils`)이 **React 렌더 경계를 넘어 1프레임 지연**으로 들어옵니다(`App.jsx:57-74` ↔ `engine.js:765-794`). 즉 코일↔접점이 한 번의 연산이 아니라 여러 렌더에 걸쳐 수렴하므로, 관측 시점에 따라 "X는 회색인데 MC1은 켜짐" 같은 **일시적 불일치 상태**가 보일 수 있습니다.

또한 코일→접점 매핑이 `engine.js`(`X_POWER→X_A1/X_A2` 등, `engine.js:870-890`)에 하드코딩되어 있고 전선 연결은 `task01.js`에 따로 있어 **이중 소스**입니다. 과제가 18개로 늘면 이 중복이 버그의 온상이 됩니다.

**해결 방향 (권장)**: 엔진을 "도달성" 모델에서 "양극(L1/L2) 노드 연결성" 모델로 재설계.
- 노드를 union-find/그래프로 묶되 접점 상태로 간선을 on/off, 코일·램프·부저는 **종단 부하**로 표시.
- 어떤 부하가 "한 단자는 L1 집합, 다른 단자는 L2 집합"에 동시에 속할 때만 동작으로 판정.
- 코일→접점 의존성은 하드코딩 대신 컴포넌트 데이터에 `controls`/`controlledBy` 필드로 선언해 task 데이터 단일 소스화.
- 수렴 루프는 `simulateCircuit` 내부에서 완전히 끝내고(이미 max 8회 루프 있음), `activeCoils`의 렌더-경계 피드백 의존을 제거.

### 제안 작업 순서 (우선순위)
1. **결함 A 먼저** — 시각 정렬이 맞아야 좌표 실측·디버깅이 신뢰 가능. 위험 낮고 효과 즉시.
2. **결함 B 엔진 재설계** — L1/L2 양극 연결성 + 코일 종단화 + 단일 소스 의존성. task01 wires의 `from/to`를 실제 1번 공개과제 단선도와 1:1 재대조.
3. **회귀 방지** — 시나리오별 기대 출력(수동 기동/정지/자기유지, 자동 FLS, EOCR 트립, 플리커)을 표로 고정해 수동 검증 체크리스트화.

**3. `src/components/PdfCanvas.jsx` — 결함 A 해소**: 패딩 없는 내부 stage 박스에 canvas와 SVG를 동일 박스로 겹침. SVG는 `inset:0; width/height:100%` + `viewBox=원본포인트`로 배율 무관 자동 정렬. `top/left:8px`·픽셀 하드코딩·잘못된 `clientWidth(패딩포함)` 스케일 제거. devicePixelRatio 반영으로 선명도 개선. 클릭 좌표는 `boundingRect` 비율로 환산.

**검증 결과(코드 트레이스)**: 자동 모드+수위감지 시 이제 X·MC1·RL이 일관되게 동작(기존 'X 꺼짐+MC1 켜짐' 모순 해소).

**남은 토폴로지 이슈(코드 아닌 데이터)**: `task01.js`에서 `T_POWER`는 `W_T_COIL`로 PB1에만 연결되어 자기유지 경로가 없음 → 버튼을 떼면 T가 소자되고, 자동 모드에선 PB1을 누르지 않으므로 T가 동작하지 않음. 실제 1번 공개과제 단선도와 대조해 T 코일의 자기유지/구동 결선을 보정해야 함(실제 도면 대조 필요, 다음 작업 권장). 엔진은 주어진 토폴로지를 충실히 시뮬레이션하므로, 이 항목은 데이터 보정 건임.
